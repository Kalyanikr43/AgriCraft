# AgriCraft Website Modification Requirements Document

## 1. Project Overview

### 1.1 Website Name
AgriCraft

### 1.2 Website Description
An innovative platform that empowers farmers to transform agricultural waste into valuable handmade products while connecting them with environmentally conscious buyers. The platform features AI-powered waste classification, product marketplace, and direct communication between farmers and buyers.

### 1.3 Modification Objectives
Simplify user flows, enhance marketplace functionality, and create a more intuitive, human-centered interface that doesn't appear AI-generated.

## 2. Homepage Modifications

### 2.1 Background Design
- Add handmade basket product images as background elements
- Maintain all other existing homepage elements
\n### 2.2 Main Navigation
Display only two primary options:
- Farmer button
- Buyer button

## 3. Farmer Flow\n
### 3.1 Waste Upload Process
- Direct farmers to upload agricultural waste image immediately after clicking'Farmer'
- Support only three waste types:\n  - Coconut shell\n  - Banana stem
  - Rice husk
\n### 3.2 Waste Classification & Guidance
- AI-powered image matching for the three supported waste types
- Display step-by-step reusable method suggestions for creating handmade products from identified waste
\n### 3.3 Product Listing Creation
After product completion, collect:\n- Handmade product image upload
- Product cost\n- Farmer's mobile number
- Other required product details

### 3.4 Automatic Marketplace Integration
- Submitted products automatically appear in the Marketplace page
- No manual approval process required

## 4. Buyer Flow

### 4.1 Direct Marketplace Access
- Clicking 'Buyer' redirects directly to Marketplace page
- No intermediate steps\n
### 4.2Marketplace Display
- Grid layout similar to Meesho e-commerce style
- Each product card shows:
  - Product image
  - Price
  - Product details
  - Materials used
\n### 4.3 Purchase Process
When buyer clicks 'Buy':\n- Option1: Display farmer's contact details for direct communication
- Option 2: Simple farmer-friendly buy flow similar to Meesho platform

### 4.4 Search & Filter
- Search by keywords\n- Filter by material type
- Filter by price range

## 5. Feedback System

### 5.1 Feedback Section
- Add feedback collection section at the end of the website
- Allow users to submit comments and suggestions
- Collect user experience feedback

## 6. Existing Features to Retain

### 6.1 User Management
- Username-based authentication system
- Role-based access control (Farmer, Buyer, Admin)\n- First registered user becomes administrator

### 6.2 Farmer Features
- Product inventory management (edit/delete)
- Waste classification history view
- Direct messaging with buyers
\n### 6.3 Admin Features
- User management dashboard
- Role assignment and modification
- Platform activity monitoring

### 6.4 Technical Features
- Automatic image compression (max 1MB, WebP format)
- Fully responsive design across all devices
- Secure authentication system

## 7. Design Requirements

### 7.1 Visual Style
- Natural earth-tone color palette: forest greens (#2d5016, #4a7c2c), warm browns (#8b4513, #a0522d), soft beige (#f5f5dc)\n- Human-made aesthetic - avoid AI-generated appearance
- Simple and natural interface design
- Clean card-based layouts with clear visual hierarchy

### 7.2 Layout Approach
- Grid-style product display similar to Meesho
- Card-based component design
- Clear visual separation between sections
- Intuitive navigation structure

### 7.3 Responsive Design
- Seamless experience on desktop, tablet, and mobile devices
- Touch-friendly interface elements
- Optimized image loading for mobile networks
